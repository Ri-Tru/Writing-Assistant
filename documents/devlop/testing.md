## 1. 测试概述

### 当前测试状态

- **测试框架**：无（纯手动测试）
- **测试数据目录**：`test/`
- **测试文件**：
  - `termTest.md` — 悬停功能测试用术语档案
  - `wordCountTest.txt` — 字数统计测试用文本文件

### 测试原则

1. 每次修改代码后，至少运行一次**回归测试清单**（见第 10 节）
2. 新增功能必须补充对应的测试场景
3. 所有测试在 **Extension Development Host** 中进行
4. 测试前执行 `Writing Assistant: 重载术语索引` 确保索引为最新状态

## 2. 测试环境设置

### 2-1. 准备工作区

在项目根目录下创建测试数据文件（或直接使用 `test/` 中的文件）。

建议的工作区结构：

```
workspace-root/
├── test/
│   ├── termTest.md              # 术语档案（核心测试文件）
│   ├── wordCountTest.txt        # 字数统计测试
│   ├── multi-file/              # 多文件测试
│   │   ├── characters.md        # 角色档案
│   │   └── locations.md         # 地点档案
│   └── edge-cases/              # 边界测试
│       ├── long-terms.md        # 长术语
│       └── special-chars.md     # 特殊字符
└── .vscode/
    └── settings.json            # 工作区设置
```

### 2-2. 基础设置

`.vscode/settings.json`：

```json
{
    "writingAssistant.term.hoverFiles": [
        "test/termTest.md",
        "test/multi-file/*.md",
        "test/edge-cases/*.md"
    ],
    "writingAssistant.term.titleLevel": 2,
    "writingAssistant.term.hoverField": "摘要",
    "writingAssistant.term.aliasField": "别称",
    "writingAssistant.term.showFields": ["性格", "实力"],
    "writingAssistant.wordCount.enabled": true,
    "writingAssistant.wordCount.excludeHeaders": true,
    "writingAssistant.wordCount.excludeBlankLines": true
}
```

## 3. 术语悬停功能测试

### 3-1. 基础测试数据

创建 `test/termTest.md`：

```markdown
## 张三
- **摘要**: 主角，初出茅庐的冒险者
- **别称**: 张老三，老张
- **性格**: 易冲动，易上头
- **实力**: C级

## 李四
- **摘要**: 主角的挚友
- **别称**: 小李
- **技能**: 剑术
- **装备**: 铁剑

## 青云山
- **摘要**: 位于大陆东部的神秘山脉，常年云雾缭绕
- **别称**: 雾山，云隐山
- **气候**: 终年湿润，多雾
```

### 3-2. 测试用例

| # | 测试场景 | 操作步骤 | 预期结果 |
|---|----------|----------|----------|
| 1 | **基础悬停** | 新建 `.md` 文件，输入"张三"，鼠标悬停在"张三"上 | 弹出悬停卡片，标题为 **张三**，内容显示"主角，初出茅庐的冒险者" |
| 2 | **中英文混合术语** | 输入 "ABC角色名123"，悬停其上 | 正则匹配成功，显示对应卡片 |
| 3 | **下划线术语** | 输入 "角色_A_1"，悬停 | 下划线被识别为有效字符，正常匹配 |
| 4 | **无匹配术语** | 输入一个不存在于 TermMap 中的词 | 无悬停卡片弹出 |
| 5 | **空档案** | 将 `hoverFiles` 设为不存在的文件或空数组，重载索引后悬停 | 无卡片弹出，控制台无报错 |
| 6 | **扫描范围边界** | 将 `scanRange` 设为 1，输入 "张三是主角"，将光标放在"三"上 | 只匹配到"三"，但如果"三"不是单独术语则无匹配 |
| 7 | **扫描范围内长术语** | 将 `scanRange` 设为 10，输入"张三是主角"，光标放在"三"上 | 向左右各扫描 10 字符，匹配到"张三" |
| 8 | **跨行术语** | 术语名包含换行前后 | 术语定义在单行标题中，跨行不匹配（预期行为） |
| 9 | **纯文本文件中悬停** | 在 `.txt` 文件中输入术语名并悬停 | 同样触发悬停卡片（HoverProvider 注册了 `plaintext`） |
| 10 | **标点符号边界** | 输入"张三。"（句号结尾），悬停在"张三"上 | 光标向前扫描不跨过标点，但仍能匹配"张三" |

### 3-3. 边界数据

创建 `test/edge-cases/long-terms.md`：

```markdown
## 在很久很久以前有一座古老的山脉它叫做青云山脉
- **摘要**: 这是一个非常长的术语名称，用于测试长术语匹配
- **别称**: 长名一号
```

创建 `test/edge-cases/special-chars.md`：

```markdown
## Alpha_1
- **摘要**: 带下划线的英文术语

## 2B
- **摘要**: 数字开头的术语

## rôle
- **摘要**: 带重音符号的术语
```

## 4. 别名匹配测试

### 4-1. 测试数据

沿用 [3-1](#3-1-基础测试数据) 中的 `termTest.md`。

### 4-2. 测试用例

| # | 测试场景 | 操作步骤 | 预期结果 |
|---|----------|----------|----------|

## 5. @hover 指令覆盖测试

### 5-1. 测试数据

```markdown
## 排除测试
<!-- @hover: exclude=true -->
- **摘要**: 这个术语不应触发悬停

## 自定义字段
<!-- @hover: hoverField=背景 -->
- **背景**: 这是一个自定义的背景描述
- **摘要**: 默认摘要字段内容（不应显示）
- **别称**: 自定一号

## 覆盖额外字段
<!-- @hover: showFields=技能,装备 -->
- **摘要**: 显示技能和装备字段
- **技能**: 剑术精通
- **装备**: 玄铁重剑
```

### 5-2. 测试用例

| # | 测试场景 | 操作步骤 | 预期结果 |
|---|----------|----------|----------|
| 1 | **exclude 排除** | 悬停在"排除测试"上 | 无悬停卡片弹出 |
| 2 | **覆盖 hoverField** | 悬停在"自定义字段"上 | 卡片显示"背景"字段内容，而非"摘要" |
| 3 | **覆盖 showFields** | 悬停在"覆盖额外字段"上 | 除摘要外，额外显示"技能"和"装备"字段 |
| 4 | **多条注释叠加** | 术语下有多个 `@hover` 指令 | 后定义的覆盖先定义的 |
| 5 | **文件级 @hover** | 在文件开头设置 `<!-- @hover: file.titleLevel=3;file.aliasField=其他 -->` | 整个文件按三级标题解析，别名字段改为"其他" |
| 6 | **无效 @hover 语法** | 格式错误的注释 `<!-- @hover wrong -->` | 忽略该行，不报错 |

## 6. 多文件与文件规则测试

### 6-1. 测试数据

创建 `test/multi-file/characters.md`：

```markdown
## 王五
- **摘要**: 神秘的剑客
- **年龄**: 28
- **职业**: 剑士
```

创建 `test/multi-file/locations.md`：

```markdown
## 月影城
- **概述**: 大陆最大的贸易城市
- **所属国家**: 月影国
- **气候**: 四季分明
```

### 6-2. 配置

```json
"writingAssistant.term.fileRules": [
    {
        "pattern": "test/multi-file/characters.md",
        "titleLevel": 2,
        "showFields": ["年龄", "职业"]
    },
    {
        "pattern": "test/multi-file/locations.md",
        "hoverField": "概述",
        "showFields": ["所属国家", "气候"]
    }
]
```

### 6-3. 测试用例

| # | 测试场景 | 操作步骤 | 预期结果 |
|---|----------|----------|----------|
| 1 | **文件规则悬停** | 悬停在"王五"上 | 显示摘要 + 年龄 + 职业 |
| 2 | **不同文件规则** | 悬停在"月影城"上 | 显示**概述**字段（非"摘要"）+ 所属国家 + 气候 |
| 3 | **同名术语覆盖** | 两个文件存在同名术语 | 以后加载的为准 |
| 4 | **glob 通配符文件名** | `hoverFiles` 中使用 `**/*.md` | 递归扫描所有 .md 文件 |

## 7. 预览与编辑命令测试

| # | 测试场景 | 操作步骤 | 预期结果 |
|---|----------|----------|----------|
| 1 | **查看完整设定** | 在悬停卡片中点击"查看完整设定"链接 | 在侧边以 Markdown 预览打开术语档案并定位到对应标题 |
| 2 | **MPE 预览回退** | 将 `previewMode` 设为 `mpe`，但未安装 MPE | 自动回退到 VSCode 内置预览（不应崩溃） |
| 3 | **编辑术语设定** | 在悬停卡片中点击"编辑术语设定"链接 | 在侧边栏打开档案文件，术语名称被选中并高亮 |
| 4 | **重载索引** | 修改术语档案后执行 `Writing Assistant: 重载术语索引` | 状态栏提示术语数量，悬停显示最新内容 |
| 5 | **保存自动重载** | 直接 Ctrl+S 保存档案文件 | 索引自动更新，无需手动重载 |
| 6 | **查看命令** | 打开命令面板，搜索 "Writing Assistant" | 显示三条命令：查看完整设定、编辑术语设定、重载术语索引 |

## 8. 字数统计功能测试

### 8-1. 测试数据

创建 `test/wordCountTest.txt`：

```
张三和李四是好朋友
他们一起去冒险
```

### 8-2. 测试用例

| # | 测试场景 | 操作步骤 | 预期结果 |
|---|----------|----------|----------|
| 1 | **基础统计** | 打开 `.txt` 文件 | 状态栏显示字数（示例文本应为 16 个中文字符） |
| 2 | **非 txt 文件** | 打开 `.md` 文件 | 字数统计不显示（状态栏隐藏） |
| 3 | **选区统计** | 选中部分文本 | 状态栏显示"选中/全文"格式，如 "8/16" |
| 4 | **实时更新** | 在文档中输入或删除字符 | 字数统计实时变化 |
| 5 | **排除标题** | `excludeHeaders: true`，插入 `# 标题` 行 | 标题行的字符不纳入统计 |
| 6 | **排除空行** | `excludeBlankLines: true`，插入空行 | 空行不纳入统计 |
| 7 | **中英混合** | 输入 "Hello 世界" | 统计非空白字符数（"Hello世界" = 7 字符） |
| 8 | **切换编辑器** | 在多个文件之间切换 | 字数统计跟随当前活动编辑器更新 |
| 9 | **禁用字数统计** | 将 `wordCount.enabled` 设为 `false` | 状态栏字数消失 |

## 9. 配置项测试

| # | 测试场景 | 操作步骤 | 预期结果 |
|---|----------|----------|----------|
| 1 | **hoverFiles 未匹配** | `hoverFiles` 中的 glob 未匹配任何文件 | 无索引，悬停不触发，不报错 |
| 2 | **titleLevel 变更** | 将 `titleLevel` 从 2 改为 3，档案中改用 `###` 标题 | 正常解析三级标题作为术语名 |
| 3 | **空 showFields** | 将 `showFields` 设为 `[]` | 悬停卡片只显示术语名和主字段 |
| 4 | **预览位置** | 切换 `openLocation` 为 `active` | 编辑术语在**当前**编辑组打开（而非侧边） |
| 5 | **动态更新** | 修改 settings.json 后执行重载命令 | 新配置生效（无需重启 VSCode） |

## 10. 回归测试清单

每次修改代码后，执行以下清单确保核心功能正常：

### 悬停功能

- [ ] 中文术语悬停能正常弹出卡片
- [ ] 英文术语悬停能正常弹出卡片
- [ ] 别名能触发与主术语相同的卡片
- [ ] 术语名有空格时仍然能匹配
- [ ] 排除的术语不触发悬停
- [ ] 修改档案文件后保存，悬停显示最新内容
- [ ] 重载命令后悬停显示最新内容

### 命令功能

- [ ] "查看完整设定"打开正确的档案和定位
- [ ] "编辑术语设定"打开档案并选中术语名
- [ ] "重载术语索引"提示术语数量

### 字数统计

- [ ] 打开 .txt 文件时显示字数
- [ ] 打开 .md 文件时隐藏字数
- [ ] 当选区变化时更新统计
- [ ] 文档内容变化时实时更新
- [ ] 禁用配置项后隐藏

### 稳定性

- [ ] 启动时无报错
- [ ] 打开非工作区项目（没有 workspaceFolders）时有提示
- [ ] 重载索引在档案文件缺失时不会崩溃
- [ ] 悬停在非术语上不会报错

## 11. 自动化测试展望

当前项目采用纯手动测试。如果将来引入自动化测试，以下是建议：

### 11-1. 推荐框架

- **[Jest](https://jestjs.io/)** — 适合纯函数单元测试（`markdownUtils.js`、`globUtils.js`）
- **[Mocha](https://mochajs.org/) + [Sinon](https://sinonjs.org/)** — 适合模拟 VSCode API 的测试

### 11-2. 可自动化测试的模块

| 模块 | 测试类型 | 建议 |
|------|----------|------|
| `markdownUtils.js` | 单元测试 | 纯函数，高优先级 — 测试标题正则、字段解析、注释解析 |
| `globUtils.js` | 单元测试 | 纯函数，高优先级 — 测试各种 glob 模式匹配 |
| `config.js` | 单元测试（mock VSCode API） | 中优先级 |
| `indexBuilder.js` | 集成测试 | 低优先级（依赖文件系统和 VSCode API） |
| `wordCount.js` | 单元测试（逻辑部分） | 中优先级 — 统计逻辑可独立测试 |
| `hoverProvider.js` | 集成测试 | 低优先级（强依赖 VSCode API） |

### 11-3. 测试文件组织建议

```
test/
├── unit/
│   ├── markdownUtils.test.js
│   ├── globUtils.test.js
│   └── wordCount.test.js
├── fixtures/
│   ├── termTest.md
│   ├── multi-file/
│   └── edge-cases/
└── integration/
    └── indexBuilder.test.js
```
